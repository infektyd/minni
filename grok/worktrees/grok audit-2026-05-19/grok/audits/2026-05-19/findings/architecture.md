## What Looks Solid

- **Centralized identity via EffectivePrincipal (G11):** principal.py:42-48 + sovrd.py:1314-1320 (every relevant handler resolves stamped principal before work; mismatch errors -32000 structured). Back-compat only via explicit principals/*.json + legacy_agent_ids. No silent caller trust.

All daemon access via sovereign.ts jsonRpcSocketRequestWithFallback; vault writes via documented plugin API (vault.ts). This is the correct separation.
- **Consistent envelope wrapping:** agent_envelope.ts:54-81 (wrapEnvelope, stableStringify, version="1", ordered keys, <sovereign:context> tags) used uniformly by hook.ts, codex-hook.ts, server.ts. SKILL.md:19 mandates correct parse discipline.
- **Contract linting + status tags:** test_g03_contract_matrix.py:14-34 + IMPLEMENTED/PARTIAL/PLANNED markers throughout AGENT/VAULT/POLICY.md (last updated 2026-04-26).
- **Handoff redaction + guardrails:** sovrd.py:234 (_HANDOFF_KINDS), 243 (_SECRET_PATTERNS + _redact_text/_redact_value), 534 (redact before write), 538-549 (G12 vault_root check post-principal), 551+ (G23 wikilink containment).
- **Vault hygiene + append-only invariants:** Multiple writers append to index.md/log.md (never rewrite); raw/ immutable intent enforced in docs + code paths.
- **MCP security improvements (G11/G13):** server.ts comments + removal of model-supplied agentId/afmPrepareUrl; stamping at handler.
- **Cross-agent recall provenance:** Results carry agent_origin (AGENT.md:75-76, retrieval code); privacy_level + can_read_document gate (principal.py:400+).

**Overall architecture posture:** Strong logical separation at the daemon/MCP boundary for the sovereign-memory plugin (no engine imports). Weaknesses concentrated in legacy OpenClaw integration (direct internals + deprecated HTTP), per-agent hardcoded wiring in plugin hooks + sovrd vault alias map, and minor doc/impl drift on vault layout + planned-vs-shipped RPC methods. Identity/handoff/vault surfaces have solid code-level contracts but thinner high-level schema docs than the core retrieval envelope. 11 surfaces, 8 leaks, ~15 agent-specific sites identified.

**End of Phase 1a findings.** Next phases should prioritize cutting the 3 backend import paths, removing agent aliases from sovrd.py:286 (make fully dynamic), and aligning VAULT.md layout exactly with VAULT_DIRS + adding full handoff/identity envelope JSON schemas to contracts/.
