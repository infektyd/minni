# 1. Overview

Minni (legacy name: Sovereign Memory) is a local-first memory system for AI coding agents. The Python engine (`src/minni/minnid.py`, `src/minni/*`) stores runtime truth in SQLite (`~/.minni/minni.db`), indexes vault Markdown into FTS5/embeddings/FAISS, and exposes recall, learning, handoff, candidate review, status, and AFM compile over JSON-RPC. The TypeScript plugin (`plugins/minni`) exposes MCP tools, agent hooks, a loopback console, vault helpers, handoff/task packets, and AFM bridge calls. `openclaw-extension/` is **not on main** — treat any still-installed copy as a separate legacy surface; `src/minni/openclaw-tool.sh` remains an in-tree helper.

The intended production boundary is the local macOS user account, not an internet-facing multi-tenant service. The primary assets are the SQLite DB, vaults (`raw/`, `wiki/`, `inbox/`, logs), principal files under `$MINNI_HOME/principals`, AFM/hook context packets, candidate learnings, and derived indexes/exports/graph dumps. Security impact is calibrated around confidentiality of private memory, integrity of durable memory, agent identity/provenance, preventing untrusted recalled text from becoming instructions, and preventing unattended promotion of hostile content into durable learnings.

Important design intent is visible in code — **do not over-credit these as complete mitigations**: Unix sockets are created under `~/.minni/run` with *intended* owner-only permissions (chmod is best-effort warn-and-continue; there is no `getpeereid`); caller-supplied `agent_id` is replaced by a server-stamped `EffectivePrincipal` on **daemon-mediated** paths only; recall results are wrapped as evidence and instruction-like chunks are flagged; `privacy_level=blocked` is always excluded; missing privacy defaults to `safe` (fail-open); learning defaults to candidate staging rather than durable writes (except opt-in AFM consolidation auto-promote); AFM output is draft/review oriented with allowlisted targets; the console binds loopback with Host/Origin/Sec-Fetch and a bearer token by default; audit/frontmatter parsing has hardening (leading YAML only; writeback rejects forged `---`).

When reviewing a PR, prefer diffs that weaken a named residual or invent a new surface that skips a trust layer. Do not treat same-uid FS access as “mitigated by `can_read_document`.”

# 2. Threat model, Trust boundaries and assumptions

**Trust layers (do not collapse):**

1. **Same-uid filesystem plane** — DB, vault markdown, inbox JSON, principals, launchd/daemon env. Daemon authz does **not** apply. Inherent residual of local-first same-uid storage. Escalation: edit daemon env / `MINNI_LOCAL_OPERATOR` to re-open wildcard `main` in strict mode.
2. **Cross-uid / socket plane** — `~/.minni/run/minnid.sock` intended 0600 in 0700 run dir. **No peer-credential check.** Chmod failure continues serving; mkdir/bind TOCTOU exists. If modes fail, any local UID gets wildcard-operator RPC.
3. **Daemon-mediated RPC** — Provenance gate + capability table. Zero-config synthesizes `main` with `capabilities=["*"]` and empty `allowed_vault_roots` (path gates vacuous). Strict mode (`principals/*.json`) constrains; `MINNI_LOCAL_OPERATOR` re-opens wildcard.
4. **Plugin-local FS writes** — `writeVaultPage` / `recordAudit` never hit `EffectivePrincipal`; safety is schema-pinned vault path only. `gate.shared` is attribution, not authorization. Daemon-down: gated tools fail closed; local vault writes proceed.
5. **Console** — Loopback-only; Host/Origin/Sec-Fetch; bearer by default (`MINNI_CONSOLE_TOKEN` or auto-gen). Residuals: token in URL query string; `/api/health` unauthenticated; `MINNI_CONSOLE_NO_AUTH=1` opt-out.
6. **Optional HTTP (`--port`)** — No auth, unbounded body, same `_dispatch` as UDS, stamped as local/UDS operator. **Critical when enabled** — not mere configuration risk.
7. **AFM / derived** — Allowlist (loopback or allowlisted HTTPS); `0.0.0.0` loopback quirk. `MINNI_AFM_LOOP` (default off) can auto-promote candidates from vault `inbox/` ingest; content-semantic gate is mainly `is_instruction_like` plus privacy/quality/dedup — not human endorsement.
8. **Deep Research (console, opt-in)** — `MINNI_CONSOLE_DEEP_RESEARCH=1` + bearer + local bind. External CLI/Python exec + internet egress; `local-docs`/`hybrid` can ship vault material to cloud.
9. **Recall / privacy** — Evidence envelopes + `instruction_like`. `blocked` always dropped. Missing privacy → `safe` (fail-open). Fail-closed requires an explicit `private` / `local-only` / `blocked` stamp.

**Attacker-controlled inputs** include JSON-RPC params from any process that can reach the daemon socket (same-uid always; cross-uid if perms fail) or optional HTTP fallback; MCP tool arguments influenced by the model or user prompts; vault Markdown, frontmatter, wikilinks, inbox/handoff packets, and audit details; AFM/generated summaries; intentionally large payloads; local webpages driving the loopback console; and **same-uid direct opens of the DB/vault files** (bypasses all RPC gates). A compromised lower-trust local process in the same user account is in scope for high-impact local abuse — including mediated socket/console abuse **and** direct FS access.

**Operator-controlled inputs** include environment variables (`MINNI_*`, legacy `SOVEREIGN_*` paths, AFM mode/targets, helper paths, `MINNI_LOCAL_OPERATOR`, `MINNI_RESOLVE_OPERATORS`, console token/`NO_AUTH`/Deep Research), principal files under `$MINNI_HOME/principals`, allowed vault roots, launchd/socket configuration, UI enablement, and manual candidate/AFM endorsement decisions. Pointing vault/DB/AFM at unsafe locations is usually configuration risk — **except** enabling HTTP fallback or Deep Research, which create Critical/High technical surfaces.

**Developer-controlled inputs** include tests, fixtures, package scripts, migrations, local CLI commands (`agent_api`, graph export), and build tooling. Direct Python CLI and graph export are primarily local operator/developer surfaces; they become security-relevant if wired to a model, UI, or network without the daemon's principal and read-policy gates. Graph export has **no privacy filter**.

**Assumptions:** OS filesystem permissions protect the user's home *when chmod succeeds*; this is single-user local software; no untrusted internet clients are expected **unless Deep Research or non-loopback AFM/HTTP is enabled**; AFM/native helpers are local unless explicitly allowlisted; and recalled memory must always be treated as evidence/citation, never as system, developer, or user instruction.

**Canonical paths (reject stale names in PRs):** `$MINNI_HOME` / `~/.minni`, socket `~/.minni/run/minnid.sock`, engine `src/minni/minnid.py`, plugin `plugins/minni`. Stale `engine/sovrd.py`, `~/.sovereign-memory/run`, or assuming `openclaw-extension/` is current on main are documentation/scope errors.

# 3. Attack surface, mitigations and attacker stories

**Daemon IPC, identity, and authorization.** `src/minni/minnid.py` is the most sensitive runtime boundary. It routes line-delimited JSON-RPC on a Unix socket, caps UDS request bodies (1 MiB), and optionally starts an HTTP fallback. `src/minni/principal.py` creates `EffectivePrincipal`, validates principal file permissions (strict vs warn), rejects reserved `agent_id` wire claims, supports `allowed_vault_roots`, and centralizes `can_read_document`. This mitigates caller-supplied identity/vault confusion **on daemon-mediated paths in strict mode**. Residual stories to flag in PRs: a same-user process calls the socket to exfiltrate/stage (or opens DB/vault directly); optional HTTP enabled without auth / unbounded body / UDS-equivalent principal stamp; zero-config wildcard `main` with `*` and empty vault roots; `MINNI_LOCAL_OPERATOR` reopen; plugin passes stale/default agent IDs; methods that omit principal or `can_read_document` checks; new transports that inherit `transport="uds"` defaults.

**Retrieval, privacy, and prompt injection.** `src/minni/retrieval.py` combines FTS, embeddings, reranking, expansion, and progressive disclosure. It filters lifecycle/privacy metadata, calls `can_read_document` when a principal is supplied, and wraps chunks in evidence envelopes with `instruction_like` from `src/minni/safety.py`. This is the core defense against prompt injection through recalled memory **when the principal is passed and metadata is honest**. Highest-risk bugs / PR regressions: semantic recall or expansion of `private`, `local-only`, `blocked`, foreign-agent, or wrong-workspace documents; `handle_read` prior-context hardcoding `privacy_level=safe` before the gate; cross-share via attacker-writable `page_type ∈ {wiki,handoff,synthesis,decision,session}` or `agent` containing `wiki`/`handoff`; `search_learnings` with no privacy dimension (`cross_agent` unlocks all); linked-context/AFM neighborhood summaries that skip the gate; hook vault search (`filterSafeVaultResults`) that relies on frontmatter heuristics instead of the daemon gate; or downstream TS packets that include private Markdown outside daemon policy. Prompt injection is Critical only when it changes authority (automatic durable learn, command execution, cross-agent handoff, or secret export); merely recalling hostile text should be mitigated by evidence framing — **except** AFM-loop auto-promote of plausible false facts that are not instruction-shaped.

**Vault, wikilinks, handoff, and path traversal.** Vault Markdown and handoff packets are untrusted content. The daemon uses vault-root guards and realpath checks for supplied vault paths and handoff wikilink references. TS vault helpers resolve references under a configured vault root (`assertUnder`); audit logging escapes forged entries. Areas to scrutinize: references that should be limited to `wiki/` but resolve elsewhere; symlink/race cases; vacuous vault-root binding under zero-config empty roots; handoff packet redaction; inbox JSON → candidate staging → consolidation; legacy OpenClaw fallback filesystem reads that join `vaultDir` with attacker-supplied relative paths if that surface is reintroduced.

**Writes and learning integrity.** Default `learn` is proposal-first: non-forced calls stage `candidate_packets`; `force=true` and cross-principal candidate resolution require an operator principal via `explicitly_allowed_operator` (bare `*` does **not** grant cross-principal resolve — credit this control). Writeback parameterizes SQL, writes provenance edges, detects contradictions, and rejects frontmatter-forging bodies before Markdown writeback. Bugs are high or critical if untrusted model text can bypass staging, impersonate another principal, supersede learnings, write vault pages without audit/review, or if **AFM consolidation auto-accepts** candidates (`resolved_by=afm-consolidation`) when the loop is enabled. Plugin-local `writeVaultPage`/`recordAudit` remain ungated by `EffectivePrincipal`.

**AFM, summaries, Deep Research, and derived surfaces.** AFM paths compile sensitive context for local/native or bridge models. Bridge calls enforce loopback or explicit allowlist and redact local paths/secrets in errors (label-oriented redaction — incomplete for JSON-quoted or bare tokens); native helpers use JSON stdin/stdout and timeouts; AFM drafts require endorsement for durable draft status — but consolidation can still auto-promote learnings when `MINNI_AFM_LOOP` is on. Risks: context exfiltration to nonlocal endpoints; prompt injection in summaries promoted as facts; private/raw/session material entering AFM prompts; HyDE (default off) posting queries to unauthenticated loopback AFM. Deep Research (console opt-in) can exec external tools and egress vault content to the cloud. SQLite remains truth; FAISS, graph exports, hook envelopes, handoff packets, and AFM drafts are derived/review surfaces and must not override policy. Graph export CLI dumps document metadata without privacy filtering — do not wire it to lesser-privileged surfaces.

**Console, hooks, and adapters.** The UI server binds to loopback, checks Host/Origin/Sec-Fetch-Site, requires JSON content type, caps bodies, and requires a bearer by default (auto-gen if unset). It can trigger candidate resolution and Deep Research subprocesses when enabled and authorized. Residuals: token-in-URL disclosure; unauthenticated `/api/health`; `NO_AUTH=1`. Hooks automatically inject context on session start/user prompt/pre-compact/stop — they must minimize private paths and honor read policy (daemon path when up; vault heuristic fallback is weaker). Do not assume legacy OpenClaw bearer/read-root behavior equals hardened `minnid`.

**Secret redaction.** Best-effort, label-only (`key: value`, PEM). Does not reliably catch `"api_key": "sk-…"` or bare high-entropy tokens. Treat as incomplete Medium control, not a floor.

**Availability and robustness.** Caps exist for daemon UDS requests, learn content, UI JSON, and result counts. HTTP body is uncapped when enabled (DoS + Critical authz). Heavy embedding/reranking, AFM, and Deep Research remain local DoS/egress targets; rate limiting is less critical for single-user local use but becomes important if sockets or UI are exposed beyond the account.

# 4. Criticality calibration (critical, high, medium, low)

**Critical:** remote or cross-account/cross-uid access to the daemon (HTTP enabled, or socket perms fail with no peercred); same-uid direct exfil/tamper of private/blocked SQLite/vault/session material (inherent accepted residual — still Critical-to-state, not “mitigated by principal”); identity/principal bypass allowing one agent/process to read/write as another and commit durable memory (including HTTP stamped as UDS operator, or wildcard `main` abused for force/learn where gates allow); path traversal or helper/AFM/Deep Research command execution that reads secrets or ships them off-box without operator intent; prompt injection / inbox poisoning that causes automatic durable writes (AFM loop on), shell execution, or secret export.

**High:** same-user lower-trust local process abuses socket/loopback console to search, expand, stage, or resolve sensitive memory (mediated path — still High even when direct FS is available); `can_read_document` / privacy bypass in semantic recall, expand/drill, export packs, AFM prep, hook context, `handle_read` hardcoded-safe meta, frontmatter-driven cross-share, or ungated learnings; vault/handoff wikilink traversal outside intended roots **in strict mode**; AFM SSRF / non-allowlisted endpoint receiving context, or allowlisted nonlocal endpoint receiving private context; Deep Research enabled with a obtainable console token → vault→cloud exfil / external exec; oversized HTTP/AFM payload causing daemon exhaustion when those surfaces are on.

**Medium:** audit-log or frontmatter injection that misleads review but does not grant authority; incomplete secret redaction; missing redaction of local paths or adapter filenames; console token-in-URL and `/api/health` info leak; UI CSRF requiring local browser quirks despite Host/Origin checks; incorrect lifecycle/status handling causing stale or rejected content to appear as evidence; contract/doc drift (e.g. claiming null privacy → `local-only`); supply-chain weakness from broad dependency ranges; graph/FAISS exports leaking metadata only when manually shared; Deep Research feature existing behind opt-in (document even when off).

**Low:** developer/test-only failures; cosmetic status/path disclosures already local to the user; denial of service requiring deliberate operator misconfiguration (other than enabling HTTP — that is Critical); malformed vault pages that are safely ignored; false positives/negatives in quality heuristics that still require manual review; HyDE default-off query-harvest residual.

**Strong controls (do not regress these without explicit justification):** `explicitly_allowed_operator` for cross-principal candidate resolve (not bare `*`); console local bind + Host/Origin/Sec-Fetch + default bearer; AFM SSRF allowlist fail-closed for credential-trick/subdomain/encoded-IP (note `0.0.0.0` quirk); reserved wire claims of `main`/`operator` denied without operator context; UDS 1 MiB line limit; evidence envelopes; leading-YAML-only frontmatter parse; handoff wikilink containment; MCP removal of model-facing `vaultPath` / `afmPrepareUrl`.

**Explicit non-goals / accepted residuals:** cryptographic agent authentication (SEC-021 deferred); protecting memory from the account owner or malware running as that user; multi-tenant remote service hardening; treating `gate.shared` as an ACL.
